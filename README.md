# 🚀 ReplyRocket — AI Revenue Engine for Social Creators

> Turn every Instagram comment, DM, and reel into revenue.

ReplyRocket is a production-ready, multi-tenant AI sales platform that:
1. Watches Instagram-style comments for keywords (e.g. "PRICE")
2. Auto-fires personalized DMs to the commenter
3. Lets a **Claude Sonnet 4.5** AI Auto-Closer qualify the lead and close the sale
4. Generates **real Razorpay payment links** in-chat
5. Marks leads as converted automatically when payment is received (via Razorpay webhook)

---

## 🎬 The "aha" flow (60 seconds)

1. Sign up → AI agent + 2 demo campaigns seeded for you
2. Edit the **AI Agent** with your real services + pricing + booking link
3. Open **IG Simulator** → comment "PRICE" as a fan
4. ReplyRocket fires the auto-DM → AI Auto-Closer takes over
5. Roleplay as the prospect in the **Inbox** → AI recommends, books, and shares a **REAL Razorpay payment link**
6. Click the link → pay with Razorpay test card → webhook marks the lead Converted with revenue logged

## 🧱 Stack

- **Next.js 14** (App Router) + **MongoDB** + **Tailwind** + **shadcn/ui**
- **Claude Sonnet 4.5** via Emergent Universal LLM key (OpenAI-compatible endpoint)
- **Razorpay Payment Links API** with HMAC-SHA256 webhook signature verification

## 📡 Razorpay Webhook setup (REQUIRED for auto-conversion)

After deploying, configure Razorpay to notify ReplyRocket when a payment is captured:

1. Sign in to your **Razorpay Dashboard** → switch to **Test Mode** (top right toggle)
2. **Account & Settings → Webhooks → Add New Webhook**
3. **Webhook URL**: `https://YOUR_DOMAIN/api/webhooks/razorpay`
4. **Secret**: paste the same value as your `RAZORPAY_WEBHOOK_SECRET` env var
5. **Active Events**: select ✅ `payment_link.paid`
6. Save — Razorpay will send a test ping

When a customer pays a link generated by ReplyRocket, the webhook will:
- ✅ Mark the lead as `converted` and bump `score: hot`
- ✅ Log revenue
- ✅ Increment campaign conversion stats
- ✅ Drop a system message into the chat thread (`💰 Payment received: ₹X`)

You can find the exact webhook URL inside the app under **Settings → Razorpay Webhook**.

## 🧠 How the AI Auto-Closer works

The AI uses **Claude Sonnet 4.5** with structured JSON output. Every reply contains:
```json
{
  "reply": "Aww Goldens are amazing 🐕 Our Full Spa is ₹1499 ...",
  "intent": "pricing|booking|info|payment|objection|smalltalk|closing|other",
  "lead_score": "hot|warm|cold",
  "lead_stage": "new|interested|qualified|negotiation|converted|lost",
  "actions": [
    { "type": "share_payment_link", "amount": 1499, "label": "Full Spa" },
    { "type": "share_booking_link" },
    { "type": "share_pricing" }
  ]
}
```

When the AI returns `share_payment_link`, the backend:
1. Calls Razorpay `POST /v1/payment_links` with the amount and conversation reference ID
2. Stores the `short_url` in the message metadata
3. UI renders a clickable green chip → opens Razorpay's hosted checkout
4. After payment → Razorpay webhook → conversion auto-logged

## 🛡️ Security notes

- All passwords are bcrypt-hashed (10 rounds)
- JWT cookies are `httpOnly`, `secure`, `sameSite=lax`, 7-day expiry
- All authenticated routes scoped to `workspace_id` (multi-tenant isolation)
- Razorpay webhook uses **HMAC-SHA256 timing-safe** signature verification
- Webhook idempotency: every `x-razorpay-event-id` is stored once; duplicate events are skipped
- Webhook endpoint reads **raw request body** before JSON-parsing (signature is over raw bytes)

## 🗺️ What's NOT in this prototype (yet)

- **Real Instagram Graph API** — requires Meta Business verification + app review (2-6 weeks). The IG Simulator on the dashboard demonstrates the exact same flow. Wiring real IG only requires:
  - `META_APP_ID`, `META_APP_SECRET`, `META_PAGE_ACCESS_TOKEN`, `META_VERIFY_TOKEN` env vars
  - A webhook endpoint at `/api/webhooks/meta` (not yet implemented — easy to add in 1 hr)
- **WhatsApp Cloud API** — same as above; needs WABA + phone number ID
- **TikTok integration**
- **Visual funnel builder** (drag-and-drop)
- **AI memory / RAG** (upload PDFs to train the agent)
- **Voice-note transcription**
- **Google OAuth login**

All can be added incrementally. The auth system, multi-tenancy, webhook scaffolding, and Razorpay flow are production-ready.

## 📞 Support

- AI is failing → check `EMERGENT_LLM_KEY` is valid
- Razorpay link not generated → check `RAZORPAY_KEY_ID/SECRET` and that the customer contact number doesn't have repeating digits (Razorpay rejects e.g. `+919999999999`)
- Webhook not firing conversion → confirm webhook secret matches in Razorpay Dashboard AND `RAZORPAY_WEBHOOK_SECRET` env var
- Page stuck on "loading" → check `JWT_SECRET` is set
